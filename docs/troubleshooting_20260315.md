# 트러블슈팅 로그 (Troubleshooting Log) - 20260315

## 1. 안드로이드 카드 프레임 잔상 문제 (Android Frame Artifacts)

- **문제점**: `borderWidth`와 `borderColor`를 사용하는 카드 커스텀 컴포넌트가 안드로이드 저사양 기기 또는 특정 해상도에서 선이 끊기거나 굵기가 일정하지 않은 잔상이 발생함.
- **원인 분석**: React Native의 View 테두리 렌더링 엔진이 안드로이드 하드웨어 가속 환경에서 일부 아티팩트를 발생시키는 것으로 판단됨.
- **해결 방안**:
  - 모든 카드 스타일에서 `borderWidth: 1`을 제거.
  - 대신 `backgroundColor`를 배경색보다 살짝 밝거나 어두운 색상(`colors.bgSecondary`)으로 가득 채워 면으로 구분하는 방식으로 변경.
  - 그림자(`shadowColor`, `elevation`) 값을 미세하게 조정하여 가독성 확보.
- **결과**: 안드로이드에서 깨끗한 UI 렌더링 확인 및 전반적인 디자인의 미니멀리즘 강화.

## 2. 번레이트 계산의 부동 소수점 오차 누적 (Floating Point Drift)

- **문제점**: `perSecondBurnRate`를 단순 float으로 계산하여 틱마다 잔액에서 뺄 경우, 시간이 지남에 따라 실제 순자산과 미세한 오차가 발생하는 현상 발견.
- **원인 분석**: JavaScript의 64비트 부동 소수점 연산(`0.1 + 0.2 !== 0.3`)의 한계로 인해 초당 수백 번 발생하는 연산에서 오차가 누적됨.
- **해결 방안**:
  - 모든 내부 번레이트 계산 엔진에 `SCALE = 1000` 상수를 도입.
  - 금액을 '밀리원' 단위의 정수로 변환하여 연산한 뒤, 최종 디스플레이 직전에만 1000으로 나누어 소수점 1자리까지 표시.
- **결과**: 장시간 앱 실행 시에도 잔액 정합성 100% 유지.

## 3. 정기 거래(Virtual Transactions) 데이터 정렬 오류

- **문제점**: DB에서 불러온 실제 거래와 루프를 통해 생성한 가상 거래를 합칠 때, 날짜 순서가 뒤섞여 리스트 하단에 가상 거래가 몰리는 현상 발생.
- **해결 방안**: Combine 및 필터링 이후 최종적으로 `transactionsData.sort((a, b) => b.date.localeCompare(a.date))`를 명시적으로 수행하여 날짜 역순(최신순) 정렬 강제.
- **결과**: 실제 거래와 예약된 가상 거래가 날짜별로 자연스럽게 섞여 출력됨.

## 4. CLI 도구 파일 접근 권한 이슈 (Permission Denied)

- **문제점**: 터미널에서 `ls`, `find` 등의 명령어로 `/docs` 등의 특정 디렉토리를 탐색하려고 할 때 `Operation not permitted` 오류 발생.
- **원인 분석**: macOS의 Sandbox 정책 또는 터미널 프로세스에 '전체 디스크 접근 권한(Full Disk Access)'이 할당되지 않아 발생하는 환경적 이슈.
- **우회 방법**:
  - 터미널 직접 명령 대신 IDE 내장 파일 탐색기 또는 AI 에이전트의 파일 처리 도구(`list_dir`, `view_file`)를 사용하여 파일 및 디렉토리 관리.
  - 시스템 설정에서 터미널(또는 VS Code)에 전체 디스크 접근 권한 부여 권장.
