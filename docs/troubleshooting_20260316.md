# 🐞 Troubleshooting Log - 2026-03-16

## 1. Donut Chart 다크 모드 시각적 결함
- **문제**: Donut Chart 중앙의 "구멍(Hole)" 부분이 다크 모드 전환 시에도 흰색으로 유지되어 배경과 이질감이 발생함.
- **원인**: `donutHole` 스타일의 `backgroundColor`가 고정된 값으로 할당되어 있었음.
- **해결**: `useAppTheme`를 통해 현재 테마의 `colors.bg`를 동적으로 주입하여 배경과 자연스럽게 블렌딩되도록 수정.


## 2. 안드로이드 그림자(Shadow) 미출력 이슈
- **문제**: iOS에서는 정상적으로 보이던 카드의 그림자가 안드로이드 기기(Note 20 등)에서 전혀 렌더링되지 않음.
- **원인**: React Native에서 안드로이드는 `shadowColor`, `shadowOpacity` 등을 지원하지 않으며 `elevation` 속성만 사용함.
- **해결**:
  - 스타일 객체에 `elevation` 추가.
  - 그림자가 잘리지 않도록 부모 뷰에 `overflow: 'visible'` 설정 및 적절한 마진 확보.

## 3. 캘린더 테마 실시간 미반영 현황
- **문제**: 설정에서 다크 모드를 변경해도 캘린더 화면으로 돌아왔을 때 기존 테마 색상이 남아있는 현상.
- **원인**: `react-native-calendars`에 전달되는 `theme` 객체가 메모이제이션되어 있어 외부 상태(isDark) 변화를 감지하지 못함.
- **해결**: `useMemo`의 의존성 배열에 `isDark`를 명시하고, 컴포넌트 리렌더링 시 테마 객체가 새로 생성되도록 강제함.

## 4. 통계 탭 데이터 오버플로우
- **문제**: 지출 카테고리가 많아질 경우 범례(Legend)가 화면 아래로 잘리는 이슈.
- **해결**: 통계 화면 전체를 `ScrollView`로 감싸고, 차트와 범례가 자연스럽게 스크롤되도록 레이아웃 재설계.
## 5. SecurityOverlay 'Rules of Hooks' 위반 에러
- **문제**: PIN 설정 중 `SecurityOverlay` 컴포넌트에서 `useCallback`, `useEffect` 등의 훅이 조건문(`if (!isSecurityEnabled || !isAppLocked)`) 뒤에 호출되어 런타임 에러 발생.
- **원인**: React의 'Rules of Hooks'에 따라 모든 훅은 어떤 상황에서도 동일한 순서대로 실행되어야 함.
- **해결**: 컴포넌트 최상단에서 모든 훅을 선언하고, UI 렌더링 직전에만 조건부 조기 리턴(`return null`)을 수행하도록 구조 개선.

## 6. PIN 설정 모달 오지표 출력 (UX 이슈)
- **문제**: 신규 PIN 설정 중 복잡성 검사(1234 등)에 실패한 후, 올바른 번호를 입력해 '확인' 단계로 넘어가자마자 'PIN 번호가 일치하지 않습니다'라는 메시지가 출력됨.
- **원인**: `errorCount` 상태가 복잡성 검사 실패 시 증가한 후, 다음 단계(confirm)로 넘어갈 때 초기화되지 않아 힌트 로직이 오작동함.
- **해결**: 복잡성 검사 통과 시 `setErrorCount(0)`를 호출하여 상태를 명시적으로 초기화하도록 수정.

## 7. PIN 확인 단계 안내 문구 모호함
- **문제**: 신규 PIN 설정 시 '확인' 단계에서 안내 문구가 첫 입력 시와 동일하게 "N자리 숫자를 입력하세요"로 나와 사용자가 현재 단계를 혼동할 수 있음.
- **해결**: `confirm` 단계에서 에러가 없을 경우 "PIN 번호를 한 번 더 입력하세요"라고 표시되도록 힌트 로직 개선.
