# 📅 Development Log - 2026-03-17

## 🚀 오늘의 주요 작업 (Summary)
오늘은 데이터 관리 시스템의 구조적 결함을 해결하기 위한 **DataService 리팩토링(SRP 적용)**과 **설정 화면 UI의 전면 개편**을 수행했습니다. '온디바이스 로컬 앱'의 정체성을 강화하고, 사용자 친화적인 설정 환경을 구축하는 데 집중했습니다.

## 🛠 작업 상세 내역

### 1. DataService 및 데이터 관리 레이어 리팩토링
- **단일 책임 원칙(SRP) 적용**: 거대했던 `DataService.ts`를 성격에 따라 3개의 계층으로 분리했습니다.
  - **`FileHelper.ts` (Utils)**: 플랫폼별(Web/iOS/Android) 파일 저장 및 공유(Sharing) 로직을 전담.
  - **`DataParser.ts` (Utils)**: JSON/CSV 데이터의 파싱 및 유효성 검증 로직을 추상화.
  - **`DataService.ts` (Service)**: DB와 유틸리티를 조율하는 얇은 서비스 레이어로 전환 (코드량 약 70% 감소).
- **Repository Pattern 도입**: DB 복원(`restoreDatabase`) 및 대량 삽입(`importTransactions`) 로직을 `sqlite.ts` 내부로 이동하여 데이터 정합성을 강화했습니다.

### 2. 설정 화면 (Settings) UI/UX 전면 개편
- **메뉴 구조 재구성**: 설정 항목을 논리적으로 그룹화하여 가독성을 높였습니다.
  - **테마**: 다크 모드(3-Way 선택: 라이트/다크/시스템) 및 색약 모드 토글 추가.
  - **자산 설정**: 정기 내역 자동 생성 및 감가 계산 자동화 옵션 통합.
  - **보안 및 개인정보**: PIN 잠금, 생체 인증, 개인정보 보호 모드 유지.
  - **데이터 관리**: 버튼 4개를 2개(내보내기/불러오기)로 통합하고 선택형 팝업(JSON/CSV) 적용.
- **데이터 초기화 기능 고도화**: 단순 샘플 로드를 넘어, 모든 데이터를 안전하게 비울 수 있는 '데이터 초기화' 기능을 추가했습니다.

### 3. 테마 시스템 고도화
- **시스템 테마 연동**: 라이트/다크 강제 전환 외에 OS 설정을 따르는 '시스템' 모드를 추가했습니다.
- **색약 모드 지원**: 색약자를 위한 고대비/색상 보정 테마를 스토어와 연동하여 실시간 반영되도록 구현했습니다.
