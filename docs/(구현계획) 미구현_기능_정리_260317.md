# Ohmymoney 미구현 기능 정리 & 스프린트 계획

---

## 📊 현황: 미구현 기능 현황판

현재 금융 핵심 기능(자산관리, 대출, 정기지출)은 거의 완성되었고, 이제 **설정 메뉴 & 편의성 개선** 단계로 진입할 때입니다.

**전체 미구현 항목: 13개** (약 90~110시간)

---

## Group A: 설정 메뉴 (5개 항목, ~16시간)

기본적인 사용자 설정 및 보안 기능. 앱의 안정성과 편의성을 높이는 필수 영역.

| #       | 기능         | 설명                         | 추정 시간 | 우선순위      | 상태    |
| ------- | ---------- | -------------------------- | ----- | --------- | ----- |
| **A-1** | 테마 & 표시 설정 | 라이트/다크 모드, 글꼴 크기, 통화 형식    | 2~3시간 | 낮음        | ⬜ 미시작 |
| **A-2** | 보안 설정      | FaceID/PIN 로그인, 생체인증       | 4~6시간 | **🔴 높음** | ⬜ 미시작 |
| **A-3** | 개인정보 보호    | 데이터 마스킹, 민감정보 숨김           | 1~2시간 | 중간        | ⬜ 미시작 |
| **A-4** | 데이터 관리     | iCloud Sync, CSV 내보내기/가져오기 | 3~4시간 | 중간        | ⬜ 미시작 |
| **A-5** | 앱 정보       | 버전, 라이선스, 피드백, 오픈소스 라이선스   | 1시간   | 낮음        | ⬜ 미시작 |

**그룹 합계:** ~16시간

---

## Group B: 편의성 & UX 개선 (5개 항목, ~22시간)

사용자 경험 향상 및 일반적인 불편함 해결. **Ziro님의 피드백 반영 필수**.

| #       | 기능                | 설명                             | 추정 시간   | 우선순위      | 상태    |
| ------- | ----------------- | ------------------------------ | ------- | --------- | ----- |
| **B-1** | 정기지출/대출 수정        | 기존 정기지출 및 대출 정보 편집 기능          | 8~10시간  | **🔴 높음** | ⬜ 미시작 |
| **B-2** | 고정지출 모니터링         | 월 고정지출 vs 예산 비율 시각화            | 4~5시간   | 중간        | ⬜ 미시작 |
| **B-3** | 고대비/색약 테마         | 접근성 향상 (빨강/초록 색약 대응)           | 3~4시간   | 중간        | ⬜ 미시작 |
| **B-4** | 앱 수익 모델           | Pro 구독 결제, 구독 관리 UI            | 2~3시간   | 중간        | ⬜ 미시작 |
| **B-5** | Dynamic Island 지원 | iOS 14.1+ Dynamic Island 통지 표시 | 10~15시간 | 낮음        | ⬜ 미시작 |

**그룹 합계:** ~32~40시간 (동적 계산)

**주요 이슈:**
- B-1은 Ziro님의 2026-03-16 피드백 반영 필수
- B-5는 시간이 많이 소요되므로 후순위 추천

## Group C: 고급 기능 (3개 항목, ~55~80시간)

AI 기반 자동화 및 고급 분석. 차별화 지점이지만 시간 투자가 많이 필요합니다.

| # | 기능 | 설명 | 추정 시간 | 우선순위 | 상태 |
|---|------|------|---------|---------|------|
| **C-1** | 지능형 자산 예측 모델 | ML 기반 월별 자산 추이 예측 | 20~30시간 | 중간 | ⬜ 미시작 |
| **C-2** | 자동 입력 (SMS/카톡) | ❌ 불가능 (프라이버시), 대신 AI 자동완성 | 30~50시간 | 낮음 | ⬜ 미시작 |
| **C-3** | 감가상각 UI | 자산 가치 변화 시뮬레이션 | 6~8시간 | 낮음 | ⬜ 미시작 |

**그룹 합계:** ~56~88시간

**주요 이슈:**
- C-2는 기술적 불가능 → AI 자동완성으로 대체하는 것이 현실적
- C-1, C-3은 병렬 처리 권장

## 📋 의사결정 체크리스트

### Q1: Sprint 1을 먼저 할까요?

**추천:** YES ✅
- 이유: 기본 설정 없이는 UX가 완성되지 않음
- 특히 보안(FaceID) + 데이터 관리(Sync)는 필수

---

### Q2: B-1(정기지출 수정)의 우선순위는?

**추천:** 즉시 진행 ⭐⭐
- Ziro님의 2026-03-16 피드백에서 강조됨
- 사용자 만족도 높음
- 복잡도 중간 (3개 개월차 처리 등)

---

### Q3: C-2(자동 입력) 기술적 불가능을 어떻게 처리?

**현재 결정:**
- ❌ SMS/카톡 자동 파싱 불가능 (앱스토어 정책)
- ✅ 대신: AI 자동완성 엔진 (사용자가 "은" 입력 → "은선물" 추천)
- 효과: 70~80% 자동화 효과 유지

---

### Q4: 병렬 처리할까요?

**추천:** 선택적
- **병렬 가능 조합:**
  - C-1 (예측 모델) + C-3 (감가상각)
  - 이유: 독립적인 기능
- **순차 필수:**
  - A → B → C (의존성 있음)
