# 상세 트러블슈팅 로그 (Expanded Troubleshooting Log) - 20260314

## 1. 신규 자산 등록 직후 "억울한" 감가상각 발생

- **증상**: 자산 등록 마법사를 통해 900만 원짜리 차량을 등록하자마자 자산 가치가 8,999,075원 등으로 미세하게 깎여서 표시됨.
- **원인 분석**:
  - **타임존 불일치**: SQLite의 `CURRENT_TIMESTAMP`는 UTC 기준인 반면, 앱의 연산은 KST(Local) 기준임. 이로 인해 등록 직후임에도 시스템은 약 9시간의 세월이 흐른 것으로 간주하여 감가상각을 적용함.
- **해결책**:
  - 시간 문자열 뒤에 `Z`를 강제 삽입하여 JavaScript가 UTC로 정확히 해석하도록 보정.
  - 상각 계산 시 밀리초 단위가 아닌 '일(Day)' 단위로 `floor` 처리를 하여, 구매 후 최소 24시간이 지나야 상각이 시작되도록 로직 수정.
  - 최종 자산 가치에 `Math.round`를 적용하여 소수점 지저분함을 제거.

## 2. 마법사 모달 내 ReferenceError (Hoisting Issue)

- **증상**: 입력을 완료하지 않은 상태에서 마법사 모달을 닫을 때 `Property 'setMonthlyPayment' doesn't exist` 등의 에러와 함께 앱 크래시.
- **원인 분석**: 
  - `const`로 선언된 상태 변수와 함수들은 호이스팅(Hoisting)되지 않음.
  - 모달 초기화 함수(`resetAndClose`)가 파일 하단에 선언된 상태 변수들에 접근하려 했으나, 초기 렌더링 시점에 정의되지 않은 상태여서 참조 오류 발생.
- **해결책**:
  - 모든 `useState` 선언부와 파생 변수(`availableCashAssets` 등)를 컴포넌트 최상단으로 이동시켜 생명주기 내내 안전한 접근 보장.

## 3. 대환/학자금 대출 시 부채 잔액 미반영

- **증상**: '기관 직접 지급' 방식의 대출 등록 시, 부채 항목은 생기지만 잔액이 0원으로 유지되거나 순자산에 반영되지 않음.
- **원인 분석**: 
  - 초기 대출 구현 시 '내 계좌 입금' 트랜잭션이 있어야만 부채가 잡히도록 설계되어 있었음. '직접 지급' 시에는 입금이 없으므로 부채의 시작점(Seed)이 누락됨.
- **해결책**:
  - 부채 초기 잔액을 0으로 설정하되, 대출 실행과 동시에 **'대출 계좌 → 외부(undefined)'**로의 `transfer` 트랜잭션을 강제 생성하여 빚의 발생 사실을 데이터베이스에 각인시킴.

## 4. 코드 편집 중 발생한 SyntaxError (stock.tsx)

- **증상**: 자산 관리 화면 진입 불가 및 `Unexpected token, expected ","` 구문 에러 발생.
- **원인 분석**: 
  - 대량의 코드 교체(`replace`) 작업 중 플레이스홀더(`... later in the file ...`)가 소스 코드에 잔류하거나 JSX 태그의 열고 닫음이 꼬임.
- **해결책**:
  - `stock.tsx` 전체 구조를 정밀 검사하여 불필요한 텍스트를 제거하고 JSX 트리 구조를 수동으로 재건하여 복구.

## 5. 수치 데이터 연산 중 NaN(Not a Number) 발생

- **증상**: 대시보드의 번레이트나 순자산이 간헐적으로 비정상적으로 표시되거나 연산 결과가 나오지 않음.
- **원인 분석**: 
  - DB에서 가져온 `REAL` 타입 데이터가 문자열로 취급되거나, 빈 입력값이 연산에 포함될 때 `undefined + number` 등으로 인해 `NaN` 발생.
- **해결책**:
  - `Number(val) || 0` 패턴을 모든 연산 및 리듀서(`reduce`) 로직에 적용하여 데이터 타입 안정성(Type Safety) 확보.

## 6. 마법사 모달 내 버튼 불균형 문제

- **증상**: 첫 번째 단계에서 '이전' 버튼이 없음에도 불구하고 '다음' 버튼이 화면 중앙이 아닌 오른쪽으로 치우쳐 보임.
- **원인 분석**: '이전' 버튼의 자리를 미리 확보하기 위해 비어있는 `View` 태그를 고정 너비로 넣어두어 레이아웃이 왜곡됨.
- **해결책**: 플레이스홀더 `View`를 제거하고, '이전' 버튼이 실제로 렌더링될 때만 '다음' 버튼에 `marginLeft`를 부여하는 조건부 스타일 로직으로 변경하여 시각적 균형 확보.

## 7. Android 실기기 하단 네비게이션 바 및 프레임 잔상 문제

- **증상**:
  - 하단 안드로이드 시스템 네비게이션 바가 앱 UI를 가리는 현상 발생.
  - 특정 기기에서 카드 테두리에 하얀색/검은색 프레임 잔상이 남는 시각적 오류 재발.
- **원인 분석**:
  - `SafeAreaInsets`가 특정 안드로이드 테마에서 완벽하게 적용되지 않아 스크롤 하단부가 잘림.
  - `borderWidth: 1`과 `elevation`이 결합될 때 안드로이드 렌더링 엔진에서 안티앨리어싱 오류로 인한 프레임 아티팩트 발생.
- **해결책**:
  - `paddingBottom` 값을 `Math.max(insets.bottom, 60)` 등으로 넉넉하게 설정하여 시스템 바와의 거리를 확보.
  - 모든 메인 카드의 `borderWidth`를 제거하고, 대신 배경색을 꽉 채우는 '솔리드(Filled)' 디자인으로 변경하여 렌더링 아티팩트 근본적 차단.
