## 아이디어 검토 (2026-03-13)

### ✅ 강점

1. **실시간 재무 가시성** - 시/분/초 단위 소비 추적은 심리적 임팩트가 강함
2. **손익분기점 자동 계산** - "하루에 얼마를 벌어야 하는가"는 극도로 실용적
3. **온디바이스 설계** - 개인정보 보호 + 100% 통제권
4. **객관성 강조** - 감정적 소비 패턴을 데이터로 드러냄

### ⚠️ 검토 사항

**1. 데이터 수집의 현실성**
- "모든 소비 기록" → 자동화 필수 (카드거래, 구독료 등)
- 현금 소비는 수동입력? → 이 부분이 병목
- 제안: 통신사/은행 API 연동 + 카테고리 자동분류

**2. "돈을 버는 시점"의 정의**
- 수입 > 소비 = 버는 시점? → 너무 단순
- 제안: 
  - 고정비 대비 순이익률 (%)
  - 투자수익률 고려
  - 월/년 단위 누적

**3. 실행 단계**
- Phase 1: 소비 기록 (엑셀/스프레드시트)
- Phase 2: 자동화 (API 연동)
- Phase 3: 대시보드 (온디바이스 앱)

### 💡 즉시 시작 가능한 MVP

```
1. 간단한 마크다운/JSON 스프레드시트로 시작
2. 주간 요약 자동 계산
3. 임계값 설정 (예: 일일 지출 >100k 경고)
```

### 기술 스택 제안

- 온디바이스: Python 스크립트 + 로컬 DB (SQLite)
- 대시보드: HTML/CSS (브라우저에서 열기)
- 자동화: cron job (수입/지출 API 호출)
